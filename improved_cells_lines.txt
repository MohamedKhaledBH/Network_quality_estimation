L10: | # | What changed | Why |
L11: |---|---|---|
L12: | 1 | **Fixed spectrogram** — now uses `np.abs(np.fft.fftshift(np.fft.fft(...)))` row-wise on the STFT matrix | `scipy.spectrogram` silently discards the imaginary part on complex input; the fix uses the full complex signal |
L13: | 2 | **Per-sample normalization** instead of global min-max | Global normalization leaks test-set statistics into training |

L17: | 6 | **MobileNetV2 two-phase fine-tuning** — Phase 1: frozen base; Phase 2: top 50 layers unfrozen at lower LR | Fine-tuning the top layers on domain-specific spectrograms consistently boosts accuracy |
L18: | 7 | **Deeper classifier head** in MobileNetV2 (256 → 128 dense, two layers) | More capacity for the feature-to-class mapping |
L19: | 8 | **Cosine decay LR schedule** for MobileNetV2 fine-tuning phase | Smoother convergence vs ReduceLROnPlateau during fine-tuning |

## Cell 8 [markdown]
L1: ## 4. Spectrogram Function ✅ FIXED
L2: 
L3: **Change 1 — Fixed complex IQ spectrogram computation.**  
L4: The previous version passed a complex array to `scipy.spectrogram`, which silently used only the real part in many scipy versions, throwing away half the signal information.  
L5: The new version uses `scipy.stft` which correctly handles complex input, then takes `np.abs()` of the result for the magnitude spectrogram.
L6: 
L7: **Change 2 — Per-sample normalization.**  
L8: The previous version did a global min-max across the entire dataset *before* splitting, which leaks test-set statistics into training. Now each spectrogram is normalized independently to [0, 1].

## Cell 9 [code]
L1: # ✅ FIX 1: Use scipy.stft which correctly handles complex IQ input
L2: # ✅ FIX 2: Normalize each spectrogram independently (no data leakage)
L3: 
L4: def fast_spectrogram(iq_signal):
L5:     """Convert an IQ sample to a log-magnitude spectrogram image.
L6:     
L7:     Steps:
L8:     1. Detect whether IQ is stored as (2, N) or (N, 2)
L9:     2. Combine I and Q into a complex signal
L10:     3. Use STFT (Short-Time Fourier Transform) on the 1D complex series
L11:     4. Take absolute value of the complex STFT output → magnitude spectrogram
L12:     5. Apply log1p to compress dynamic range
L13:     6. Normalize PER SAMPLE to [0,1] to avoid data leakage
L14:     7. Resize to 128x128 pixels for the CNN
L15:     """
L16:     iq_signal = np.asarray(iq_signal)
L17: 
L18:     if iq_signal.ndim == 1:
L19:         complex_signal = iq_signal.astype(np.complex64)
L20:     elif iq_signal.ndim == 2:
L21:         if iq_signal.shape[0] == 2 and iq_signal.shape[1] != 2:
L22:             complex_signal = iq_signal[0] + 1j * iq_signal[1]
L23:         elif iq_signal.shape[-1] == 2:
L24:             complex_signal = iq_signal[:, 0] + 1j * iq_signal[:, 1]
L25:         else:
L26:             raise ValueError(f'Unexpected IQ sample shape: {iq_signal.shape}')
L27:     else:
L28:         raise ValueError(f'Unexpected IQ sample rank: {iq_signal.ndim}')
L29: 
L30:     complex_signal = np.asarray(complex_signal).reshape(-1)
L31: 
L32:     # Keep STFT parameters valid even if the sample orientation changes
L33:     nperseg = min(64, complex_signal.size)
L34:     noverlap = min(48, max(0, nperseg - 1))
L35: 
L36:     # ✅ scipy.stft correctly handles complex-valued signals
L37:     _, _, Zxx = stft(complex_signal, nperseg=nperseg, noverlap=noverlap)
L38: 
L39:     # Magnitude spectrogram (absolute value of complex STFT)
L40:     Sxx = np.abs(Zxx)
L41:     Sxx = np.log1p(Sxx)                    # log-magnitude compression
L42: 
L43:     # ✅ Per-sample normalization — no leakage between train/test
L44:     s_min, s_max = Sxx.min(), Sxx.max()
L45:     if s_max > s_min:
L46:         Sxx = (Sxx - s_min) / (s_max - s_min)
L47: 
L48:     return cv2.resize(Sxx.astype('float32'), (128, 128))

## Cell 36 [markdown]
L1: ## 14. Model 3 — Transfer Learning with MobileNetV2 + Fine-Tuning ✅ MAJOR UPGRADE
L2: 
L3: **Changes 6, 7, 8 applied here:**
L4: 
L5: - **Change 6 — Two-phase fine-tuning:** Phase 1 trains only the new classifier head (base frozen). Phase 2 unfreezes the top 50 layers of MobileNetV2 and fine-tunes them at a much lower learning rate. This adapts the pretrained features to spectrogram data.
L6: - **Change 7 — Deeper classifier head:** Two Dense layers (256 + 128) instead of one, giving more capacity to map MobileNetV2 features to the 3 quality classes.
L7: - **Change 8 — Cosine decay LR schedule during fine-tuning:** Provides a smooth, gradual learning rate reduction during Phase 2, preventing catastrophic forgetting of pretrained weights.

## Cell 37 [code]
L1: def build_transfer_model(input_shape):
L2:     """Transfer Learning: MobileNetV2 + two-phase fine-tuning.
L3:     
L4:     Phase 1 (this cell): base frozen, train classifier head only.
L5:     Phase 2 (next cell): top 50 layers unfrozen, fine-tune at low LR.
L6:     """
L7:     base_model = tf.keras.applications.MobileNetV2(
L8:         input_shape=(128, 128, 3),
L9:         include_top=False,
L10:         weights='imagenet'
L11:     )
L12:     base_model.trainable = False  # Frozen for Phase 1
L13:
